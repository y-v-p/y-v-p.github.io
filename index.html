Naive demo of classification of K+ --> pi + pi0 events in LKr using ConvNetJS MNIST demo

Description

This demo trains a Convolutional Neural Network on an augmented dataset generated from a single example event extracted from a presentation given by A. Ceccucci here. An example of a K+ --> pi+ pi0 is shown in LKr detector:

After removing all the labels and lines, the cleaned event becomes:
with the size of 700x700 pixels

One can now scale it down to a smaller size suitable for neural network processing. The MNIST demo uses 28x28 pixels. For a quick demostration the same size is chosen. While for a real study the events will be generated by a Monte Carlo, here to generate a large dataset quickly one can use a python library for augmenting image datasets. ImgAug. The python script to generate the batch datasets can be found here The picture below shows an example of 10x10 grid with generated events by use of random rotations, translations with some gaussian noise added and dead channels.

To make the demonstation quicker the total number of images is chosen to be 10000 split into batches of 500 hundred.

Once all the dataset is loaded, the convolution neural network is trained on the train subset of the data and then gives prediction and measures accuracy on a test set. The webpage with the neural network runs automatically once loaded and only uses javascript. The performance and characteristics of the network is shown below. With quick experimenting, it was found that for this demo setting number of catergories to 3 is optimal for convergence. After a few minutes of running the network should show good convergence and examples at the bottom of the page show that only one category is selected as expected from a rather articial input dataset.
